# 19 — Implementation Checklist

## Before any task

- [ ] Identify active sprint.
- [ ] Read matching `.agents/rules`.
- [ ] Read navigation map.
- [ ] Read sprint plan section.
- [ ] Read domain docs.
- [ ] Confirm scope and out-of-scope.

## During implementation

- [ ] Keep change small.
- [ ] Do not mix unrelated UI/backend/schema/AI changes.
- [ ] Use typed data.
- [ ] Avoid fake production logic.
- [ ] Add TODO only when intentionally deferred.
