# 02 — User Personas and Entry Flows

## Persona A — User 0 Writing Skill

Pain:

- tidak tahu mulai dari mana,
- takut form panjang,
- tidak paham istilah premise/arc/trope/POV,
- hanya punya rasa cerita,
- butuh diarahkan seperti ngobrol.

Flow:

```txt
Aku belum punya ide
→ Chat Story Agent
→ Agent menangkap rasa cerita
→ 3 konsep cerita
→ pilih konsep
→ Fondasi Cerita otomatis
→ Outline 10 bab
→ Tulis Bab 1 per adegan
```

UI default harus memakai bahasa sederhana:

- Story Bible → Fondasi Cerita
- Reveal Gate → Jadwal Rahasia
- Beat → Adegan
- Canon → Fakta Terkunci

## Persona B — User Punya Ide Kasar

Flow:

```txt
Aku punya ide kasar
→ paste ide 1 paragraf
→ AI buat 3 versi pengembangan
→ pilih arah
→ Fondasi Cerita
→ Outline
→ Tulis Bab
```

## Persona C — User Sudah Punya Draft

Flow:

```txt
Aku sudah punya draft
→ paste/upload draft
→ AI ekstrak karakter, fakta, timeline, voice
→ user review Fondasi Cerita hasil ekstraksi
→ lanjutkan cerita
```

## Persona D — Advanced Writer

Flow:

```txt
Aku ingin kontrol advanced
→ edit Fondasi Cerita
→ edit Jadwal Rahasia
→ inspect Context Packet
→ override validator bila perlu
→ version history
```

Advanced Mode harus opt-in. Jangan jadikan default.

## Persona E — Repair Only

Flow:

```txt
Aku hanya mau cek cerita
→ paste bab
→ AI cek plot hole, dialog, cliffhanger, mobile readability
→ user pilih perbaikan
```
