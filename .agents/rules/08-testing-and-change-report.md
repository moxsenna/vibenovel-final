# VibeNovel Agent Rule — Testing and Change Report

After modifying code, report:

- files changed,
- what was implemented,
- what was not implemented,
- typecheck result,
- lint result if available,
- test result if available,
- manual verification steps,
- known limitations,
- next recommended step.

A feature is not complete just because the UI exists.
It must persist data correctly, respect business rules, and pass acceptance criteria when the sprint requires persistence.

Sprint 1 exception:
UI can use dummy data, but it must not pretend to be production persistence.
